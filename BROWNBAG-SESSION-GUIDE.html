Architecture — How It Works
  • Foundation: Playwright (Microsoft's modern browser automation)
  • Layer 1: Adapters — Web, API, SAP, Mobile (unified interface)

Slide 4

8:00 – 11:00 (3 min)
Key Feature: Page Object Model (POM)
  • BasePage class — every page object inherits self-healing, visual, discovery
  • Locators defined once → used everywhere
  • If Google changes their search box selector → only 1 file to update
  • Enterprise-ready pattern used by all major test teams worldwide

Code example: 

class GoogleHomePage extends BasePage {
  readonly searchBox = this.first("textarea[name='q']");
  readonly logo = this.first("img[alt='Google']");

  async typeSearch(query: string) {
    await this.searchBox.fill(query);
  }
}
Show the before/after: without POM, selectors scattered across 50 test files. With POM + BasePage, change one file and all tests continue working. This is the maintainability story management cares about.

Slide 5

11:00 – 14:00 (3 min)
Key Feature: Self-Healing Selectors
  • Problem: Dev changes id="btn-submit" to id="submit-button" → test breaks
  • Solution: Atomiq AI tries 4 healing strategies automatically:
  •    1. Attribute fallback (data-testid, role, aria-label)
  •    2. Text content matching
  •    3. Structural similarity (DOM position, siblings)
  •    4. AI-assisted (LLM analyzes the page to find the element)
  • Result: Tests keep running even after UI changes — zero manual fix
This is the "wow" feature for both audiences. For management: "This means your CI pipeline doesn't break on every sprint." For devs: "You don't spend Monday morning fixing 30 broken selectors."
Animation or diagram: Selector breaks → 4 strategies tried → test heals itself → green checkmark.

Slide 6

14:00 – 17:00 (3 min)
Key Feature: Pluggable AI Providers
  • 4 LLM providers supported: OpenAI GPT-4o, Azure OpenAI, Google Gemini, Anthropic Claude
  • AI generates tests from natural language descriptions
  • AI diagnoses failures — root cause + suggested fix
  • AI enhances tests — adds assertions, accessibility checks
  • Switch providers with one config change — no lock-in
// Switch AI providers at runtime — zero code change
ai.engine.setActiveProvider("claude");
ai.engine.setActiveProvider("openai");
Emphasize no vendor lock-in. If the company switches from OpenAI to Azure (for compliance), it's a one-line config change. For technical audience: mention it uses dynamic imports so packages are optional peer dependencies.
