How Agents Communicate

MessageBus Pattern

  • Pub/Sub — agents subscribe by role
  • Request/Reply — await responses
  • Broadcast — system-wide events
  • Full message history for debugging

Agent Lifecycle

  • Register → Start → Execute → Report
  • AgentRegistry manages discovery
  • Supervisor routes tasks automatically
  • Parallel or sequential execution
Supervisor → MessageBus → WebAgent WebAgent executes tests → MessageBus → Supervisor receives result { type: "result", metrics: { passed: 5, failed: 0 } }

Key Features

🔧 Self-Healing Selectors

  • 4 strategies: attribute, text, structure, AI
  • Auto-fix broken selectors at runtime
  • Zero manual maintenance

🤖 Pluggable AI Providers

  • OpenAI, Azure OpenAI, Gemini, Claude
  • Switch with one config change
  • AI generates tests from descriptions

📊 Visual Regression

  • Pixel-level screenshot comparison
  • Auto-baseline management
  • Mask dynamic regions

📋 Page Object Model

  • BasePage with built-in healing
  • Locators defined once, used everywhere
  • Enterprise maintainability pattern
